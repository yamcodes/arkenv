:::important
This plugin requires [ArkType](https://arktype.io) to be installed.
:::

The ArkEnv Vite plugin integrates directly into Vite's build pipeline to validate your environment variables against your schema. If a variable is missing or fails validation, the plugin will halt the build or dev server immediately with a clear error message.

By matching Vite's default security model, only variables prefixed with `VITE_` (along with `NODE_ENV` if included in your schema) are exposed to your client bundle. The plugin handles the underlying compiler configuration so you get complete autocomplete and typesafety when accessing `import.meta.env` across your project.

## Quickstart

The easiest way to get started is with the [ArkEnv CLI](/docs/cli). It automatically configures `@arkenv/vite-plugin` for your existing Vite project, or spins up a new one from a template.

```npm
npx arkenv@latest init
```

<Accordions>
  <Accordion title="Manual Installation">
    ### Installation [step] [!toc]

    ```package-install
    @arkenv/core @arkenv/vite-plugin arktype
    ```

    :::note
    `@arkenv/vite-plugin` lists `@arkenv/core` as a **peer dependency**. This ensures that the engine resolves to a single root-level instance across your codebase, preventing "dual package hazards" that can break runtime type evaluations and singleton context.
    :::

    For a Standard Schema setup (e.g., Zod, Valibot) without ArkType:

    ```package-install
    @arkenv/standard @arkenv/vite-plugin zod
    ```

    ### Configure plugin [step] [!toc]

    Define your schema as an exported constant in `vite.config.ts`. This allows the schema to be imported for typesafety in other files.

    ```ts title="vite.config.ts" twoslash
    import arkenv from "@arkenv/vite-plugin";
    import { type } from '@arkenv/core';
    import { defineConfig } from "vite";

    export const Env = type({
    	PORT: "number.port",
    	VITE_MY_VAR: "string"
    });

    export default defineConfig({
    	plugins: [arkenv(Env)]
    });
    ```

    ### Logging [step] [!toc]

    Pass an optional third argument to configure build-time logging. Use `logLevel: "silent"` to suppress warnings, or provide a custom `logger` object:

    ```ts title="vite.config.ts"
    export default defineConfig({
      plugins: [
        // Pass `undefined` when you only need logging options
        arkenv(Env, undefined, { logLevel: "warn" }),
      ],
    });
    ```

    ### Add typesafety [step] [!toc]

    To make `import.meta.env` fully typesafe, add the following to `src/vite-env.d.ts`:

    ```ts title="src/vite-env.d.ts"
    /// <reference types="vite/client" />

    type ImportMetaEnvAugmented = import("@arkenv/vite-plugin").ImportMetaEnvAugmented<
    	typeof import("../vite.config").Env
    >;

    interface ImportMetaEnv extends ImportMetaEnvAugmented {}

    interface ImportMeta {
    	readonly env: ImportMetaEnv;
    }
    ```

    :::tip
    For more details on advanced typing options, see [Typing import.meta.env](/docs/vite-plugin/typing-import-meta-env).
    :::

    ### Use in your code [step] [!toc]

    You can now use `import.meta.env` with full typesafety and autocompletion throughout your application.

    ```tsx title="src/App.tsx" twoslash
    // @noErrors
    // @filename: vite-env.d.ts
    /// <reference types="vite/client" />
    type ImportMetaEnvAugmented = {
    	VITE_MY_VAR: string
    }
    interface ImportMetaEnv extends ImportMetaEnvAugmented {}
    interface ImportMeta {
    	readonly env: ImportMetaEnv
    }

    // @filename: App.tsx
    // ---cut---
    const apiUrl = import.meta.env.VITE_MY_VAR;
    //      ^?

    // Autocomplete works out-of-the-box!
    const typing = import.meta.env.
    //                             ^|
    ```
  </Accordion>
</Accordions>

With this setup, if any environment variable is missing or invalid, your dev server won't start and your production build will fail with an error:

```bash title="Terminal"
ArkEnvError: Errors found while validating environment variables
  VITE_MY_VAR must be a string (was missing)
  PORT must be an integer between 0 and 65535 (was "hello")
```

## Using Standard Schema validators (Zod, Valibot, etc.)

This plugin uses ArkType for validation, which natively supports Standard Schema. You can freely mix ArkType DSL strings with Zod or Valibot validators in the same schema - no extra configuration needed.

```ts title="vite.config.ts" twoslash
import arkenv from "@arkenv/vite-plugin";
import { type } from '@arkenv/core';
import { defineConfig } from "vite";
import { z } from "zod";

export const Env = type({
	VITE_API_URL: z.string().url(),
	VITE_API_KEY: z.string().min(1),
	VITE_DEBUG: "boolean"
});

export default defineConfig({
	plugins: [arkenv(Env)]
});
```

```tsx title="src/App.tsx" twoslash
// @noErrors
// @filename: vite-env.d.ts
/// <reference types="vite/client" />
type ImportMetaEnvAugmented = {
	VITE_API_URL: string
	VITE_API_KEY: string
	VITE_DEBUG: boolean
}
interface ImportMetaEnv extends ImportMetaEnvAugmented {}
interface ImportMeta {
	readonly env: ImportMetaEnv
}

// @filename: App.tsx
// ---cut---
const apiUrl = import.meta.env.VITE_API_URL;
//      ^?

// Autocomplete works out-of-the-box!
const typing = import.meta.env.
//                             ^|
```

### ArkType-free mode (Standard Mode)

If you want to use Zod, Valibot, or other Standard Schema validators **without** installing `arktype` in your Vite project:

1. Install `@arkenv/standard` and `@arkenv/vite-plugin`:

```package-install
@arkenv/standard @arkenv/vite-plugin zod
```

2. Configure the plugin and schema in your Vite config using `@arkenv/vite-plugin/standard`:

```ts title="vite.config.ts"
import arkenv from "@arkenv/vite-plugin/standard";
import { defineConfig } from "vite";
import { z } from "zod";

export const Env = {
  VITE_API_URL: z.string().url(),
  VITE_API_KEY: z.string().min(1),
};

export default defineConfig({
  plugins: [arkenv(Env)]
});
```

3. Augment your types using `@arkenv/vite-plugin/standard` in `src/vite-env.d.ts`:

```ts title="src/vite-env.d.ts"
/// <reference types="vite/client" />

type ImportMetaEnvAugmented = import("@arkenv/vite-plugin/standard").ImportMetaEnvAugmented<
  typeof import("../vite.config").Env
>;

interface ImportMetaEnv extends ImportMetaEnvAugmented {}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```

For advanced workflows, see [using ArkEnv in Vite config](/docs/vite-plugin/arkenv-in-viteconfig).
