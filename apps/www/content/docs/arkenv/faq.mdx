## Why do I need ArkEnv?

We've seen teams struggle with environment variables in JavaScript projects. Your environment is such a core part of your application and yet we often avoid validating it because we don't have time for it or we think it's too much work.

We built ArkEnv to solve this problem for you. With a simple `npx @arkenv/cli@latest init` command, your project will become **bulletproof** against environment variable misconfigurations.

Read our full pitch in the [intro](/docs/arkenv) section.

P.S.

You can also [build your own environment variable validator in about 3 lines of code](/docs/arkenv/comparison#diy), but ArkEnv provides a lot more out-of-the-box, including framework integrations, automatic coercion, and LLM integration. All in a 2kb core package.

## Is ArkEnv only for Vite and Bun?

ArkEnv is designed to be **completely agnostic**. We officially support and provide first-class toolkits for:

- **Node.js** (Servers, CLIs, Scripts)
- **Bun**
- **Vite**

Because it is built on standard JavaScript APIs, ArkEnv *should* work anywhere JavaScript runs, including Deno or Edge Functions (like Cloudflare Workers). However, these environments are not yet part of our official test suite.

If you're using ArkEnv on a platform not listed above and run into issues, please [let us know on GitHub](https://github.com/yamcodes/arkenv/issues)!

## Do I have to use ArkType?

Not at all. While ArkEnv has native, zero-config support for [ArkType](https://arktype.io/) (and we think it's the best way to use it!), we are fully compatible with any **Standard Schema** validator.

This means you can use:
- **Zod**
- **Valibot**
- **Typia**
- ...and any other library that implements the [Standard Schema](https://standardschema.dev) specification.

See [arkenv/standard](/docs/arkenv/standard) for more details.

## Does ArkEnv work with Zod?

Yes! ArkEnv works seamlessly with Zod.

See our [Zod example](https://github.com/yamcodes/arkenv/tree/main/examples/with-zod) or use the [ArkEnv CLI](/docs/cli) which can set up ArkEnv + Zod for you automatically.

## Does ArkEnv work with Valibot?

Yes! ArkEnv works with any Standard Schema validator, including Valibot. The [ArkEnv CLI](/docs/cli) can set up ArkEnv + Valibot for you.

## Does ArkEnv work with Typia?

Yes! ArkEnv works with any Standard Schema validator, including Typia. Read our [Standard Schema guide](/docs/arkenv/integrations/standard-schema) for more details.

## Does ArkEnv load my `.env` files?

**No.** ArkEnv is a **validator**, not a **loader**.

Most modern frameworks (Vite, Bun, Next.js, Remix) already have built-in support for loading `.env` files. If you are using plain Node.js, we recommend using a dedicated loader like [dotenv](https://www.npmjs.com/package/dotenv) or [tsx](https://tsx.is/).

ArkEnv takes whatever is in `process.env` (or your chosen source), ensures it matches your schema, and applies transformations like [automatic coercion](/docs/arkenv/coercion).

## Can I use ArkEnv with Next.js?

No. Well, technically yes, but not officially...

We've seen some users who've had success using the core `arkenv` package in Next.js. However, this is **not** an official solution and support is not guaranteed. If you are looking for a Next.js-specific solution with built-in client/server separation, we recommend checking out [T3 Env](https://env.t3.gg).

We're looking into providing more official integrations for more frameworks in the future—keep an eye on our [roadmap](https://github.com/yamcodes/arkenv/issues/683)!

## Can I use ArkEnv with Nuxt?

Same answer as Next.js above.

## Why use ArkEnv instead of just Zod?

Using "raw" Zod for environment variables often leads to a lot of boilerplate:
1. **Manual Coercion:** Environment variables are always strings, so you have to manually `z.coerce.number()` every time. ArkEnv does this automatically.
2. **Verbose Errors:** Zod errors are designed for complex APIs. ArkEnv's errors are formatted specifically for terminal output.
3. **Framework Nuances:** In tools like Vite, variables are statically replaced at build time. ArkEnv's plugins handle this "magic" for you so you don't have to think about it.

## Is it safe to use in production?

Yes. ArkEnv is a very simple library (2kB gzipped) and has **zero external dependencies**. It's designed to defer most of the work to your existing validator and standardize environment variables specific flows. This is why it's fast and reliable - your application will fail fast if your environment is misconfigured. Install ArkEnv if you wish to prevent cryptic runtime errors.

While the API is still in v0 (see [v1 roadmap](https://github.com/yamcodes/arkenv/issues/683)), the core functionality is in place, fully tested, and is already being used in production. It is not expected to meaningfully change.
