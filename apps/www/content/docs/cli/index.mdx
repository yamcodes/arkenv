The `@arkenv/cli` is the fastest way to get started with ArkEnv. It provides an interactive walkthrough to help you choose a validation library, configure your environment schema, and set up your project structure.

## Quickstart

You don't need to install the CLI globally. You can run it directly using your favorite package manager:

```bash
pnpm dlx @arkenv/cli@latest init
```

*Or using `npx`:*

```bash
npx @arkenv/cli@latest init
```

## Features

- **Interactive Prompts**: A guided experience powered by `@clack/prompts`.
- **Zero Dependencies**: Lightweight and fast, it won't clutter your `node_modules` after use.
- **Validation Library Support**: Choose between [ArkType](https://arktype.io), [Zod](https://zod.dev), and [Valibot](https://valibot.dev).
- **Automatic Detection**: Smartly detects your runtime (Node, Bun) and framework (Vite, Solid-Start) to pre-select the best defaults.

## Commands

### `init`

The `init` command is the primary entry point for the CLI. It will:

1. **Detect Environment**: Check for existing `.env` files and package manager configurations.
2. **Select Schema Provider**: Ask you which validation library you'd like to use.
3. **Configure Options**: Ask if you'd like to enforce strict mode, where to put your configuration file, and more.
4. **Generate Code**: Create a `env.ts` (or `env.js`) file with a working schema and ArkEnv setup.

## Options

The CLI supports several flags to skip prompts or customize the behavior:

| Flag | Description |
| --- | --- |
| `--provider <name>` | Pre-select the validation provider (`arktype`, `zod`, `valibot`). |
| `--strict` | Force strict mode on the generated schema. |
| `--path <path>` | Specify the output path for the generated configuration. |
| `--yes`, `-y` | Skip all prompts and use defaults. |
